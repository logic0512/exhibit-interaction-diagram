# 展项交互图构图模式

## 作用

生成前先选一种主构图模式。构图模式决定一张图如何组织“人 - 触发 - 反馈 - 结果”。

一次只选一种主模式。不要把屏幕、投影、声音、灯光、机械、地面触发全部塞进一张图。

构图模式不是装置形态模板。墙面、地面、展柜、通道、转角、桌面、模型、特殊设备和穿戴/手持道具都可以成为主装置。不要把“展项装置”默认画成空间中央的展台。

## 1. 墙面屏幕反馈

适合：大屏、互动墙、信息墙、选择结果、数据生成、多人观看。

画法：

- 人站在屏幕前或地面站位点上。
- 手势、触摸或站位是主触发。
- 墙面屏幕出现反馈结果。
- 用少量虚线、箭头或蓝色发光连接动作和屏幕变化。

避免：

- 画成复杂 UI。
- 屏幕太多，像信息看板。
- 人只是站在屏幕前观看。

参考 example：`assets/examples/01-wall-screen-interaction.png`

## 2. 投影映射

适合：投影到墙面、桌面、模型、实物、地面或半透明介质。

投影映射不是固定构图。它有多种目标介质和空间组织方式：

- 物体贴合型：投到球面物体、标本、片状媒介、器物、机械模型或展品局部，适合强调“内容长在物体上”。
- 桌面/模型型：投到桌面、沙盘、地形、流域、城市模型，适合地图、路径、层级和网络。
- 地面/路径型：投到地面、台阶或行走路径，适合踩踏、行走和空间路线。
- 半透明/背投型：投到纱幕、膜、透明屏或背投屏，适合复原、漂浮层、剖面和轻量独立装置。
- 墙面/幕布型：只在需要较大画面、群体观看或远距离阅读时使用。

画法：

- 投影源必须可见：吊装投影机、墙洞、顶部设备或明确隐藏源。
- 先从内容选择目标介质，再决定投影源位置和光轴。
- 先确定投影机镜头朝向，再把目标面放在镜头前方；不要先在画面右侧补墙面结果。
- 投影路径必须从投影源沿镜头轴线到目标面。
- 目标面要有合理介质并面向光束：墙面/幕布、桌面、模型、地面、纱幕、半透明屏或物体表面。
- 人物动作是触发，不是投影光路。
- 反馈落在目标表面上。
- 如果外部幕布不是内容表达的最佳介质，优先把反馈放回物体、模型、桌面、地面或半透明介质上。

避免：

- 把人物手势箭头画成投影光路。
- 投影机朝向和投影落点不一致。
- 镜头朝左/朝下/朝观众，却把投影内容放到右侧墙面。
- 为了让内容清楚，固定在右侧墙上画一个投影画面。
- 所有投影类图都变成左侧操作装置、右侧独立幕布。
- 本来可以投到物体、模型、桌面或装置本体，却被改成旁边外接显示屏。
- 只有光效，没有投影源和落点。
- 复制参考投影设备造型。

参考 example：`assets/examples/02-projection-mapping.png`

## 3. 灯光反馈

适合：灯光点亮、颜色变化、区域照亮、氛围反馈、儿童低位互动。

画法：

- 人物先做一个明确触发动作。
- 灯具或灯带可识别。
- 光束、光区或被照亮区域承担反馈。
- 需要儿童视角时，明确成人和儿童身高差。

避免：

- 灯光只是装饰。
- 画成舞台效果图。
- 光效淹没人物动作。

参考 example：`assets/examples/03-lighting-feedback.png`

## 4. 声音点位

适合：声源定位、口述记忆、环境音、站位聆听、方向性声音。

画法：

- 人站进声音点位或靠近声源。
- 声音方向用波纹、声束或弧线表达。
- 人物身体朝向或手势体现正在听。
- 可以加小屏或墙面反馈，但声音关系是主角。

避免：

- 画成耳机广告。
- 声源位置不清。
- 声波只是装饰线条。

参考 example：`assets/examples/04-sound-listening-point.png`

## 5. 地面触发路径

适合：踩踏、路径点亮、空间路线、儿童探索、站位触发。

画法：

- 脚下触发点要和人物脚对齐。
- 地面路径或点位被点亮。
- 反馈终点可以是墙面、低位屏幕、模型或空间装置。
- 儿童展项要匹配儿童眼高和手高。

避免：

- 地面图案太复杂。
- 路径和反馈终点断开。
- 儿童被画成绘本角色。

参考 example：`assets/examples/05-floor-trigger-path.png`

## 6. 操作台 / 实物触发

适合：放置道具、扫描卡片、旋钮、滑杆、样本触发、实体输入。

画法：

- 手部动作必须清楚。
- 道具、触发区、反馈屏或反馈区域要形成一条链。
- 操作台可以有造型，但不能抢过玩法。
- 屏幕内容只保留反馈块，不画真实 App。

避免：

- 画成标准控制台。
- 道具和触发区没有接触。
- 按钮、旋钮、屏幕堆太多。

参考 example：`assets/examples/06-tangible-operation-table.png`

## 7. 机械响应

适合：曲柄、把手、滑块、开合结构、模型运动、实体装置联动。

画法：

- 人手连接机械输入件。
- 用箭头或残影表达运动方向。
- 结果以实体变化为主：打开、升起、旋转、移动、联动。
- 局部放大框可以少量使用，但不能变成工程图。

避免：

- 只画机器造型，不画人怎么触发。
- 画成工业蓝图。
- 反馈全靠屏幕，丢掉机械动作。

参考 example：`assets/examples/07-mechanical-response.png`

## 8. 多人协作

适合：多人输入、亲子协作、群体选择、共同完成、系统汇总。

画法：

- 人数通常 2-3 人即可；如果用户明确希望 4-5 人参与，必须给每个人分配不同阶段、不同参数、不同节点或不同任务，避免变成人群围观。
- 每个人动作不同，但都指向同一个共享结果。共享结果要表达内容意义，而不只是总灯光亮起。
- 用少量线条把输入汇总到中央或墙面反馈。
- 儿童只作为身高版本，不另起卡通风格。

避免：

- 人群过多。
- 每个人触发不同设备，画面变成设备清单。
- 共享结果不明显。

参考 example：`assets/examples/08-multi-person-collaboration.png`

## 9. 空间局部 / 墙地融合

适合：空间本身承担互动，装置不以独立展台为主，例如墙体嵌入、地面路径、展柜局部、通道、转角、台阶、半透明隔断、建筑界面上的局部互动。

画法：

- 先画清人和空间界面的关系：站位、触摸、经过、靠近、转动、观看或聆听。
- 反馈贴合对应界面，例如墙面局部显现、地面路径点亮、展柜旁信息出现、台阶节点触发、透明层叠加。
- 只补必要的传感器、边框、出光口、屏幕缝或嵌入结构，避免变成设备清单。
- 适合小空间、通道、展柜旁、沉浸房间和轻量改造场景。

避免：

- 为了让装置显眼，额外在空间中央加一个无关展台。
- 墙面/地面反馈没有内容意义，只是氛围光。
- 空间界面太大，人物动作和反馈关系被淹没。

## 选择顺序

生成前按这个顺序判断：

1. 用户创意里最核心的参与动作是什么。
2. 反馈主要发生在哪里。
3. 最适合的主构图模式是哪一种。
4. 哪些设备必须出现，哪些会干扰。
5. 是否需要查看对应 example 做低频校准。

如果选不出主构图模式，先不要生图，回到 `idea-expansion.md` 补全玩法。
