# QA Checklist

## 必过项

- 最终产出是一张清晰的装置交互图。
- 有人物。
- 生成前已选择并查看 1 张 `assets/visual-system/atomic/human/` 人物资产作为硬锁参考。
- 主人物正在参与核心玩法，不是装饰或单纯比例尺。
- 所有人物都沿用同一人物 IP：空白椭圆头、无头发、无五官、简化上衣长裤、灰黑线稿、浅灰阴影。
- 成人和儿童只在身高、姿态、朝向和动作上变化，不出现不同角色设计。
- 人物动作、触发点、反馈区域之间有视觉连接。
- 能看出装置或空间主体。
- 能看出反馈发生在哪里。
- 能看出反馈内容是什么，不是空屏幕、空投影或固定蓝色 UI。
- 屏幕、投影、灯光、声音或机械反馈里的内容与主题和玩法有关。
- 如果用户没有指定具体城市、建筑、品牌、地点或人物，画面不能自行生成真实城市地标、品牌标识、真实建筑名称或可识别专属对象。
- 如果用户指定的是设备上的小屏幕，反馈内容必须出现在该小屏幕上，不能被换成独立大屏、悬浮信息卡或方案说明框。
- 内容类型和载体匹配，例如地图/路径适合桌面、墙面、地面或模型；时间轴/照片适合墙面、屏幕或音频点位；实物展品适合展示柜旁信息或局部放大。
- 载体选择和空间大小、亮度、观看距离、人流方式匹配。
- 装置形态符合展陈空间秩序：普通互动优先独立轻量、家具化或局部安装，不无故使用顶天立地结构。
- 如果画面出现全高轨道、龙门架、全高立柱、天花吊架或大型桁架，必须能从用户需求或功能必要性解释它为什么不可替代。
- 设备工作距离成立：投影、探照灯、扫描灯、定向声源、感应器等和目标面之间有合理空隙、路径或检测范围。
- 人体可达范围成立：人物站位、手臂长度、儿童/成人身高和操作点高度匹配。
- 硬件类型和展陈设备逻辑成立：投影有光路，商显有边框/厚度，拼接屏有拼缝，LED 有观看距离意识，传感器有检测范围，声音有听音位置。
- 组件是按需出现，不是固定组件堆叠。
- 如果出现屏幕、投影、灯光、感应、声音等，能一眼识别载体类型，但没有硬复制参考样式。
- 如果出现投影，必须看出投影源、投影路径和投影落点；不能把触发联动箭头误标成投影路径。
- 如果用户描述“把手放入识别装置/识别仓/感应装置”，必须看出开放识别仓、凹槽、框口或半透明检测体积，而不是普通触摸台面。
- 操作台、机械件、实体道具可以有创意造型，但必须看出人物怎么使用。
- 如果展项面向儿童或亲子，人物身高、触摸高度、视线高度与儿童使用场景匹配。
- 画幅服务玩法，不因为默认横版而损害装置表达。
- 有 1 个明确主构图模式，而不是多个设备类型混在一起。
- 投影类图仍保持浅底方案草图感，只让投影面或局部投影区变暗，不整张变成暗场渲染。
- 3 秒内能理解“怎么玩”。
- 标注少、短、贴近动作或反馈。

## 失败信号

出现以下情况，重生成或局部编辑：

- 人物只是站在旁边。
- 去掉人物后玩法没有变化，说明人物没有承担动作。
- 人物没有使用已选的人物 atomic 资产作为统一风格锚点。
- 人物出现头发、五官、真实表情、复杂衣服、裙装、背包、帽子、明显性格化穿搭或写实游客感。
- 同一张图里成人、儿童或旁观者像不同插画系统。
- 完成图 example 的人物风格覆盖了人物 atomic 资产。
- 投影看不出从哪里投到哪里。
- 投影路径从人物、手势、按钮或操作台出发，而不是从投影源出发。
- 探照灯、扫描灯、投影机或定向设备贴着目标墙面，却声称向该墙面投射、扫描或照射。
- 设备和目标面之间没有可见工作距离、光路、投射路径或检测范围。
- 人物手臂明显过长，或身体站位离操作点太远。
- 人物够不到把手、按钮、屏幕、道具、滑轨、机械件或高处控制点。
- 儿童操作点按成人高度设计，或儿童需要越过展台/护栏/墙体操作。
- 可移动设备没有可触达的低位把手、滑块、手轮、拉绳、平衡臂或控制机构。
- 明亮空间里默认使用普通投影作为主反馈，却没有遮光、高亮投影或局部暗区说明。
- 小空间里使用大型投影、大展台或多人围合装置，空间尺度不成立。
- 为了表现上下移动、扫描、瞄准或调节，默认画成顶天立地轨道、全高立柱、龙门架、大型吊架或桁架。
- 可做成独立装置、立式台座、桌面设备、短支臂、手摇/旋钮/转轴机构的互动，被画成建筑级重结构。
- 设备体量压过展墙、展品或内容本身，破坏展陈空间秩序。
- 互动桌没有围合空间，或被放在狭窄通道里。
- 展品空间里的屏幕过大，导致实物展品变成无关背景。
- 环幕、影院或沉浸投影没有观看/站位区和可控光条件。
- 地面投影承载长文本、复杂时间轴或难以阅读的内容。
- 屏幕或投影只是通用蓝色界面，看不出主题内容。
- 用户没有指定具体城市、建筑、品牌或地点，但画面自行出现真实城市地标、品牌标识、真实建筑名称或可识别专属对象。
- 用户指定小屏幕，但画面生成了独立信息面板、底部说明条、悬浮 UI 卡或额外大屏来承载主要反馈。
- 用户要求把手放入识别装置，但画面变成普通触摸屏、按钮台、刷卡台或封闭盒子，看不出手进入检测空间。
- 硬件被画成魔法发光物，例如没有投影源的投影、没有边框厚度的商显、完全无缝的拼接屏、近距离粗颗粒 LED 承载细文字、没有视场的 Kinect/雷达、没有听音位置的音响。
- 明亮空间仍使用低亮度投影作为主反馈，却没有遮光、高亮短焦或局部暗区。
- 地踩 LED 或地面投影承载长文本、复杂表格或细密地图。
- 灯光只表现氛围，不表达状态、路径、选中、解锁或结果。
- 声音只画音符或声波，看不出声音内容和声源关系。
- 感应看不出检测范围。
- 灯光只是装饰，不承担反馈。
- 屏幕内容太像真实 App 或复杂 UI。
- 操作台和人物手部没有连接，或被画成无关的标准控制台。
- 机械件被画成固定盒体，丢失创意造型。
- 儿童展项里人物全是成人尺度。
- 竖向装置被硬塞进横版导致玩法不清。
- 画面像设备清单。
- 画面像组件组图、分格说明图或 contact sheet。
- 画面像某张 example 的变体，而不是当前创意。
- 为了套用 example，加入了当前玩法不需要的设备。
- 完成图 example 的设备造型、标注位置或主题内容被直接复制。
- 为了适配主题，人物被改成生态科普、儿童插画、商业渲染或场景插画里的普通人物。
- 画面混入多个无关动作，例如同时扫描、投票、录音、踩踏。
- 画面像海报、广告、PPT、UI、技术图纸或游戏插画。
- 投影类图整体过暗、过像舞美效果图或完成渲染图，削弱方案草图统一性。
- 画面很好看，但看不懂互动机制。
- 文字解释太多，靠读文字才能理解玩法。

## 迭代方法

- 玩法不清：删掉无关组件，放大人物动作和反馈区域。
- 人物弱：让人物手、脚、视线、身体朝向直接连接触发点。
- 人物风格漂移：先重新选择并查看 1 张人物 atomic 资产，在提示词中明确锁定空白椭圆头、无发型、无五官、统一简化服装；必要时降低 example 权重或不使用 example。
- 反馈弱：加强屏幕变化、投影落点、灯光变化、声波或机械运动。
- 内容弱：把反馈内容改成主题相关的生成结果、隐藏层、路径关系、比较结果、叙事片段、状态变化或实体结构变化。
- 内容越界：未指定真实地点或品牌时，改成泛化城市轮廓、编号建筑、抽象品牌中性对象或简化信息卡。
- 空间不匹配：根据空间大小和亮度换载体，或调整装置尺度、观看距离、站位和安装条件。
- 设备距离不成立：把投影/探照灯/扫描灯从贴墙改成前置支架、地面轨道、吊臂、天花轨道、立柱或独立灯具；画出设备到目标面的空隙和光路。
- 装置形态过重：优先改成独立落地台座、斜面操作台、带底座小设备、手摇/旋钮/短滑轨/转轴、可移动底座、桌面装置或墙前轻型支架；只有大型空间或明确功能必要时才保留全高结构。
- 人体可达不成立：移动人物站位，降低控制点，增加可触达的把手/滑块/手轮/拉绳，或把直接操作改成低位控制高位设备。
- 放入式手势识别不清：把普通台面改成开放识别仓、凹槽、U 形框口或半透明检测腔，并画出手部进入、检测范围和运动轨迹。
- 太像设备图：减少技术细节，增加观众动作和空间关系。
- 太像海报：去掉大标题、强装饰和戏剧化背景，回到方案草图。
- 太像 UI：简化屏幕内容，只保留反馈块和少量符号。
- 太复杂：一张图只保留一个核心玩法。

## 交付判断

高质量图应该先让人看见“人在怎么操作”，再看见“装置如何反馈”。如果第一眼只能看见漂亮空间或设备造型，但不知道怎么玩，就不合格。
