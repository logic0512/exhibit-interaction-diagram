# 交互信息内容

## 作用

展项交互图不只画玩法，也要画清互动之后出现的内容信息。

屏幕、投影、灯光、声音、气味、机械反馈上的内容不能变成固定样式。它们必须根据主题、玩法和触发动作设计，让人看出“观众触发后获得了什么信息、看见了什么变化、理解了什么关系”。

## 生成前必须判断

先补全这 4 件事：

- 内容主题：这件展项要表达什么知识、故事、关系、状态或结果。
- 内容载体：内容出现在屏幕、投影、灯光、声音、气味、机械、实体模型还是空间路径上。
- 内容变化：互动前后发生什么变化，例如生成、显现、比较、连接、放大、解锁、点亮、分支。
- 内容颗粒度：最终图只画关键画面，不画完整界面、完整文章或复杂系统。
- 内容动作关系：观众的动作具体改变了哪个内容变量，例如时间、位置、对象、层级、连接、参数、顺序、拆解、重组或协作状态。

如果只知道“屏幕显示内容”，但不知道显示什么，要先补全内容逻辑，不要画空屏或通用 UI。

如果只知道“触发后出现内容”，但不知道这个动作为什么能帮助理解内容，也要先补全互动逻辑。合格的内容互动不是把内容放到屏幕上，而是让动作参与内容表达。

如果内容已经作为说明、参数或原理拆解完整摆在装置上，也不能直接生图。合格互动必须让观众通过操作获得信息：动作让内容显现、改变、比较、验证、推进或汇总。去掉人物动作后仍然完整成立的信息，应该改成静态展板或说明图，不应该冒充互动装置。

## 内容交互有效性

生成前问一句：观众不做这个动作，就是否无法理解这层内容关系？

有效的内容交互通常满足以下至少一项：

- 选择：观众选择某个对象、区域、年代或选项，反馈显示对应信息。
- 连接：观众把两个或多个节点连接起来，反馈显示关系、路径、传播或因果。
- 推进：观众沿路径、时间轴、轨道或阶段移动，反馈显示过程变化。
- 拆解/重组：观众分解或组合对象，反馈显示结构、层级或系统组成。
- 调节：观众改变参数，反馈显示结果差异或系统状态。
- 扫描/定位：观众移动光斑、屏幕、探头或身体位置，反馈显示对应局部信息。
- 协作：多名观众分别完成不同输入，反馈汇总成一个共同结果。

无效或偏弱的内容交互：

- 按一下按钮，墙上出现一张与动作无关的说明图。
- 多个人分别点亮不同设备，但每个反馈只是独立信息卡。
- 内容已经完整摆在画面里，互动只是在旁边制造光效。
- 环境光、信号强度、系统状态、生态指标等被做成孤立参数牌，观众只能看，不能操作它们影响结果。
- 原理拆解被直接贴在装置表面，人物动作不改变任何层级、路径、状态或成功结果。
- 主题是演化或流程，但画面没有同一个对象的前后变化链。

## 过程型内容

遇到“演化、发展、流转、传播、通信、迁徙、生态循环、能量转换、工艺流程、系统协作”等主题时，优先使用“同一内容对象跨阶段变化”的画法。

可用对象：

- 同一条消息：文字 -> 编码 -> 波形 -> 光信号 -> 数据包 -> 重组结果。
- 同一条路线：起点 -> 节点 -> 分支 -> 汇合 -> 目的地。
- 同一件展品：外观 -> 局部 -> 内部层 -> 修复前后。
- 同一组生态数据：区域 -> 物种 -> 栖息地 -> 保护措施 -> 恢复状态。
- 同一个模型：整体 -> 拆解 -> 部件 -> 功能关系 -> 重组。

过程型内容不要只画多个独立热点。必须让人看出“同一个内容对象被互动推进”。

## 内容表达类型

### 1. 生成结果

适合：AI 生成、个人化结果、创作输出、留言记录、照片/声音转化。

画法：

- 输入动作和生成结果要对应。
- 屏幕或墙面上出现一个明确结果块。
- 可以用轮廓、图形、简短词语表达结果，不画复杂 UI。

不要画成固定仪表盘。

### 2. 隐藏层显现

适合：内部结构、历史层、地下层、微观层、材料剖面、不可见数据。

画法：

- 触发前的实体或空间要可见。
- 触发后在屏幕、投影或模型表面显现隐藏层。
- 内容最好叠加到对应对象上，而不是孤立漂浮。

不要只画蓝色光效。

### 3. 路径与关系

适合：城市路线、迁徙、流程、关系网络、传播路径、能量流动。

画法：

- 用线、点、节点或路径表达关系。
- 人物动作触发某条路径点亮或某组关系连接。
- 关系要有方向或结果，不要像装饰纹样。

不要堆太多节点。

### 4. 比较与选择

适合：投票、立场选择、参数实验、方案比较、因果后果。

画法：

- 画出选择动作。
- 反馈区出现两到三个差异结果。
- 用少量分支、色块或并置画面表达差异。

不要画成完整数据 dashboard。

### 5. 叙事片段

适合：口述史、人物故事、城市记忆、档案、文学、声音内容。

画法：

- 内容可以是声音波形、照片轮廓、档案卡片、短句碎片或空间影像。
- 重点是观众在何处触发、听见或看见哪类内容。
- 屏幕/投影上只放 1-3 个内容片段。

不要把故事写成长段文字。

### 6. 状态反馈

适合：灯光状态、设备状态、完成进度、协作汇总、系统平衡。

画法：

- 用点亮、颜色变化、进度环、轮廓亮起、机械位置变化表达状态。
- 状态必须和人物动作产生因果关系。
- 状态反馈要面向现场观众：让观众知道自己的操作是否成功、哪里改变了、下一步该怎么做。不要把状态画成给设计者看的系统诊断。

不要让灯光或图形只是好看。

### 7. 地图 / 地理 / 空间数据

适合：城市规划、游客中心、区域产业、交通、生态分布、历史路线。

画法：

- 内容可以出现在桌面、地面、墙面或大屏上。
- 地图不只是底图，要有被触发后点亮的区域、路径、节点、建筑或关系线。
- 如果是城市或区域主题，优先画“区域被选中 -> 信息卡/路径/数据层出现”。
- 如果用户没有指定具体城市、地标或建筑名称，不要自动套用真实城市符号、真实地标或可识别城市天际线。应使用泛化城市轮廓、编号地标、抽象建筑剪影或“代表建筑 A/B/C”。
- 多人围绕桌面地图时，要画清每个人的输入如何影响同一张地图。

不要把地图画成普通装饰底纹。

### 8. 时间轴 / 历史片段

适合：历史展、人物故事、企业发展、地方记忆、事件演进。

画法：

- 时间轴可以在墙面、地面路径、长屏或桌面上出现。
- 互动动作应选中某个年代、事件或人物片段。
- 反馈内容可以是照片、档案卡、短句、影像片段或高亮年份。

不要把完整时间线都写满；只画被触发的关键段。

### 9. 展品 / 展示柜 / 实物补充信息

适合：文物、产品、模型、样本、器物、工艺材料。

画法：

- 媒体内容要贴近实物：展示柜旁的小屏、投影叠加、局部放大、结构说明、材料层级。
- 观众动作可以是触摸、靠近、扫描、选择展品或打开局部信息。
- 信息内容要指向具体展品，不要漂浮成无关屏幕。

不要让屏幕抢走实体展品的主次关系。

### 10. 问答 / 学习任务

适合：儿童教育、科普判断、环保选择、公共议题、知识测验。

画法：

- 反馈内容可以是题目卡、选项块、结果提示、正确/错误状态或下一步任务。
- 儿童展项要使用大按钮、大图形、少文字、低位屏幕。
- 答题结果应连接到灯光、屏幕或实体变化。

不要画成完整答题 App。

## 按载体表达内容

### 屏幕内容

屏幕可以显示图像、数据、选择结果、生成结果或操作反馈，但必须服务主题。

要求：

- 有主题相关的抽象图形，而不是固定蓝色面板。
- 只画关键内容块，不画完整 UI。
- 能看出互动前后变化，例如新图形出现、关系连线生成、结果卡片弹出。
- 如果用户明确指定“设备上的小屏幕、手持屏、探照灯小屏、道具屏、嵌入式屏幕”，反馈内容必须留在该设备屏幕里，不能被偷换成独立大屏、悬浮信息卡、方案说明框或外置显示墙。
- 儿童展项的屏幕内容要更大、更低、更少文字。
- 城市/区域/交通类屏幕优先显示地图、路径、节点和信息卡。
- 历史/人物/文化类屏幕优先显示照片、档案卡、时间片段和短句。
- 科学/工业/生态类屏幕优先显示结构层、流程层、参数变化或系统状态。

### 投影内容

投影内容必须贴合投影目标。

要求：

- 投影到墙面：可以是时间层、场景影像、关系图、人物故事片段。
- 投影到模型：可以是路径、剖面、能量流、结构层。
- 投影到地面：可以是路径、脚下反馈、范围、流动方向。
- 投影到物体：内容要沿物体形态变化，不要像普通屏幕贴图。
- 环幕或大投影适合叙事场景、环境模拟和群体观看，不适合作为复杂操作界面。
- 地面投影适合路径、范围、地图和脚下状态，不适合放长文本。

### 灯光内容

灯光也可以传达信息，不只是照明。

要求：

- 颜色、亮灭、强弱、路径、范围代表某种状态。
- 被照亮区域要有内容意义，例如选中、解锁、完成、危险、路径、层级。
- 高亮区域不能盖住人物动作。

### 声音内容

声音内容要能被画面理解。

要求：

- 用声源、声波方向、站位点和小内容反馈说明声音来自哪里。
- 需要表现声音类型时，用短标注或波形，例如“口述”“环境音”“提示音”。
- 口述史和人物故事要把声音点位与照片、档案卡或场景片段连接起来。
- 多声源空间要画清当前人物听到的是哪个声源，不要同时打开所有声音。
- 不要只画音符。

### 气味内容

气味内容不能只靠“香味”两个字。要让观众看出气味和主题信息的关系。

要求：

- 用闻香口、气味样本、香味胶囊或气味模块说明气味从哪里来。
- 气味反馈要和选择、识别、比较、记忆或环境状态绑定，例如花朵结构对应香型、材料来源对应气味、场景记忆对应味道。
- 画面中要有轻量视觉证据：样本高亮、气味波纹、短标签、小屏结果或物体状态变化。
- 不要把气味画成充满空间的大烟雾、蓝色光束或纯装饰气流。
- 气味通常适合和屏幕、灯光或实体样本组合，但这些辅助反馈不能把图变成说明面板。

### 机械/实体内容

机械和实体也能承载信息。

要求：

- 开合、转动、升降、拼合、点亮本身就是内容变化。
- 结构变化要和主题关系有关，例如打开隐藏层、连接关系、放大内部、比较状态。

## 内容与载体配对

优先用内容类型反推载体：

- 地图/区域/路线：桌面互动、地面路径、大墙屏、模型投影。
- 时间轴/人物故事：墙面长屏、照片墙、音频点位、档案卡反馈。
- 实物/样本/工艺：展示柜、小屏、扫描、局部投影、机械开合。
- 参数/系统状态：操作台、模型、屏幕结果、灯光状态。
- 儿童学习任务：低位屏、按钮、地面点位、实体道具、物理反馈。
- 沉浸叙事：环幕、投影、声音、座位区或站位区。

如果内容类型和载体不匹配，优先换载体。例如口述记忆不一定需要大屏，地理路线不一定需要复杂 UI，实物工艺不一定需要大投影。

## 禁止

- 不要把所有屏幕都画成同一种蓝色 UI。
- 不要把投影都画成无主题的蓝色光面。
- 不要把气味只画成烟雾或香气装饰。
- 不要只画设备亮了，却看不出亮起的内容是什么。
- 不要把完整知识点、参数、流程或原理说明直接贴在装置上，当作“已经有交互信息”。
- 不要把观众无法操作的环境光、信号强度、系统状态等画成独立说明模块；要改成可操作挑战或即时反馈。
- 不要用大段文字解释内容。
- 不要把小屏幕反馈放大成独立信息面板，除非用户明确要求做局部放大框。
- 不要为了表达内容把图变成 PPT 信息图。
- 不要让内容效果和人物动作断开。
- 不要在用户未指定具体城市、品牌、建筑或地点时，自行生成真实城市名称、真实地标、品牌标识或可识别专属建筑。

## Runtime 使用

本文件只负责反馈内容判断，不提供可复制的最终生图短句。生成时把主题、反馈内容、互动后获得的信息和必要玩法证据填入 `runtime-prompt.md` 的 `Interactive information`。
