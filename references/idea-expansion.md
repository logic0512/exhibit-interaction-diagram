# 想法扩展与动作翻译

## 作用

当用户只有主题、内容方向、空间对象或一句模糊创意时，先把想法转成可互动、可画清楚的玩法，再进入生图。

本文件只用于组织互动动作、玩法机制和表达逻辑；不要复制具体案例、物件、构图、行业名或题材视觉。

## 生图前最小信息

生图前至少要补全：

- 参与者：成人、儿童、亲子、学生、专业观众、多人协作或围观者
- 参与动作：摸、踩、靠近、挥手、转动、拿取、放置、扫描、聆听、观察、协作、试做
- 触发点：按钮、道具、地面点位、感应范围、操作台、机械件、声音、视线、选择结果
- 反馈区域：屏幕、墙面投影、灯光、声音、机械运动、空间变化、实体道具、路径点亮
- 反馈内容：屏幕、投影、灯光、声音或机械反馈里具体出现什么信息，例如生成结果、隐藏层、路径关系、比较结果、叙事片段或状态变化
- 体验结果：生成、解锁、变化、比较、连接、转化、沉浸、决策、记忆留下
- 空间条件：小/中/大空间，明亮/半暗/黑场，近距离/远距离，人流是停留还是通行
- 图的用途：方案提案、甲方沟通、内部讨论、制作说明、灵感探索

如果缺 3 项以上，先进入辅助表达或创意扩展。不要急着生图。

如果缺少空间条件，先按中等亮度室内展厅保守推断；但当方案依赖投影、黑场、大展台、大范围地面互动或多人围合时，需要先补充空间假设，避免生成不成立的装置。

## 玩法机制库

扩展想法时，不要只问“用什么设备”，先选 1 个主玩法机制，再配 1 个反馈机制。单张图里通常只保留 1-2 个机制，最多 3 个，避免变成设备清单。

同时要判断反馈内容和空间匹配：

- 内容决定屏幕、投影、灯光、声音里出现什么。
- 空间决定能不能用这个载体，以及载体应该多大。
- 玩法、内容和空间三者不匹配时，优先调整载体，而不是硬画原设备。

### 1. 角色进入 / 任务身份

把观众变成某个角色，给他一个任务，而不是让他站着看资料。

适合：

- 儿童探索、职业体验、科学实验、城市寻访、侦探/考古/研究主题
- 需要把长内容变成“旅程”的项目

可画动作：

- 领取任务卡、登机牌、研究员证、探索护照
- 跟随虚拟伙伴或空间线索进入不同站点
- 完成一个小任务后点亮下一段内容

画面必须显示：人物身份或任务道具、起点、下一步反馈。

### 2. 实物操作 / 道具触发

用手里的实物作为触发介质，把内容变成可拿、可放、可翻、可拼、可扫描的动作。

适合：

- 城市记忆、工艺材料、产品样本、档案故事、儿童教育
- 需要避免纯屏幕操作的展项

可画动作：

- 放置卡片、样本、地标、产品、物种、工具
- 翻开档案、抽屉、书页、证言卡
- 拼合碎片、纹样、地图、结构件

画面必须显示：手部动作、道具位置、触发区、反馈结果。

### 3. 身体进入 / 现象体验

让观众用身体尺度进入风、水、速度、温度、压迫、距离、声音等现象。

适合：

- 自然生态、运动速度、老化体验、灾害教育、空间议题
- 抽象概念需要变成身体感受时

可画动作：

- 站入风场、水流、光区、声场、速度区
- 骑行、投球、推动、攀爬、穿过狭窄空间
- 触摸真实材料或感受温度、重量、震动

画面必须显示：身体位置、现象范围、感官反馈。

### 4. 参数实验 / 变量反馈

把知识点变成一个可改变变量、立刻看到结果的实验。

适合：

- 能源、气候、水循环、工业生产、材料性能、科学原理
- 需要让观众理解因果关系时

可画动作：

- 转动控制轮、拉动滑杆、调节光照/水量/速度/温度
- 选择变量组合，观察系统变化
- 操作实验台，看到结果生成

画面必须显示：变量控件、变化路径、结果区域。不要只画参数 UI。

### 5. 空间路径 / 站位叙事

用行走、站位、路线选择和空间距离表达时间、关系和立场。

适合：

- 城市街区、历史时间线、迁徙路线、社会议题、地方记忆
- 内容本身有地理、时间、路径或立场差异时

可画动作：

- 踩亮地面路径
- 站到不同位置触发不同声音或影像
- 走过时间廊、街区地图、关系网络

画面必须显示：人物脚下点位、路径方向、空间反馈。

### 6. 多视角 / 选择判断

让观众做选择或比较，看到不同立场、后果或解释并置。

适合：

- 社会议题、公共决策、历史争议、城市更新、可持续议题
- 不应该给唯一答案的内容

可画动作：

- 投票、选择立场、匹配观点
- 比较两个方案或两条路径
- 把事件、人物、影响后果连接起来

画面必须显示：选择动作、分歧反馈、不是单一正确答案。

### 7. 声音定位 / 口述记忆

用声音承载人物、记忆、环境和过程，让空间变成可听的叙事层。

适合：

- 城市记忆、文学、音乐、历史证言、生态环境、工业过程
- 图像不好表达“人声、氛围、过程”的内容

可画动作：

- 站到某个声音点听见人物故事
- 戴耳机移动，声音随位置变化
- 触发物件后出现对应环境音或口述史

画面必须显示：声源位置、人物站位、声波方向或声场范围。

### 8. 模型激活 / 机械响应

把系统、结构或过程做成可被触发的实体模型和机械响应。

适合：

- 工业流程、城市基础设施、交通系统、机械结构、生态模型
- 需要让复杂系统一眼可见时

可画动作：

- 转动曲柄、推动滑块、打开剖面
- 点亮模型节点，触发结构运动
- 控制小风车、水泵、齿轮、传送带或动态模型

画面必须显示：人如何操作模型、模型如何运动、反馈在哪里发生。

### 9. 影像映射 / 层级显现

用投影、屏幕或映射把隐藏层、时间层、数据层显现出来。

适合：

- 内部结构、历史叠加、地下/水下/微观世界、数据网络
- 需要把不可见内容变成可见层时

可画动作：

- 扫描部件后显示内部结构
- 触摸节点后墙面展开时间层
- 投影叠加到模型或实物表面

画面必须显示：触发动作、显现区域、影像和实体之间的对应关系。

### 10. 协作任务 / 群体完成

把单点观看改成多人分工、共同完成或共同见证的体验。

适合：

- 航空调度、能源系统、城市治理、团队任务、亲子协作
- 内容强调系统协同或群体选择时

可画动作：

- 多人站在不同点位共同触发
- 一人操作，一人观察，一人选择
- 多个输入汇总成同一个大反馈

画面必须显示：人物分工、多个触发点、共同结果。

### 11. 创造输出 / 留下结果

让参与者生成一件结果，形成个人化记忆或可带走的表达。

适合：

- 品牌体验、儿童创作、文学声音、工艺体验、城市记忆
- 需要让观众“留下自己的痕迹”时

可画动作：

- 录音、留言、合影、生成角色、打印标签
- 选择材料或纹样后生成个人作品
- 把个人输入汇入公共墙面

画面必须显示：输入动作、生成结果、结果出现在哪里。

### 12. 尺度转换 / 微观宏观

把看不见、太大或太小的内容换成身体可理解的尺度。

适合：

- 微生物、人体结构、机械内部、城市系统、地质时间、宇宙尺度
- 内容本身缺少身体尺度时

可画动作：

- 站在巨大剖面或微观放大结构前
- 通过观察孔、显微镜、扫描区看见隐藏层
- 操作模型后看到内部被放大显示

画面必须显示：人物尺度、被放大的对象、尺度变化的反馈。

## 机制组合方法

一个成熟展项通常不是“一个设备”，而是一个主机制加一个反馈机制：

- 角色进入 + 空间路径：把参观变成任务旅程。
- 实物操作 + 影像映射：用道具触发隐藏内容。
- 身体进入 + 声音定位：用移动和站位触发环境记忆。
- 参数实验 + 模型激活：用变量改变实体系统。
- 多视角选择 + 创造输出：让观众做判断并留下结果。
- 协作任务 + 大屏/墙面反馈：把多人输入汇总成公共结果。
- 尺度转换 + 模型激活：把不可见结构变成可操作实体。

生成图时，优先画清主机制。辅助机制只服务于解释反馈，不要抢走画面。

## 抽象概念到互动动作

### 城市与地方记忆

常用动作：

- 走入街区路径，脚下点亮地点记忆
- 触摸地图节点，墙面展开人物故事
- 翻动旧照片、档案页或城市切片
- 选择年代，空间灯光和影像切换
- 聆听口述史，声音从对应方位出现
- 放置地标道具，触发城市关系网络

适合反馈：地图投影、墙面时间轴、地面路径、声音点位、模型点亮、街区影像叠加。

优先机制：空间路径 / 站位叙事、声音定位 / 口述记忆、实物操作 / 道具触发、创造输出 / 留下结果。

### 自然生态与循环系统

常用动作：

- 放置物种、资源或环境变量卡片
- 踩踏水流、迁徙或能量路径
- 转动循环轮，看到系统变化
- 调节光照、水量、温度或风向
- 靠近不同生态位，触发局部反馈
- 多人协作维持系统平衡

适合反馈：流动投影、路径点亮、生态网络生成、灯光颜色变化、声音环境变化、机械模型运动。

优先机制：参数实验 / 变量反馈、身体进入 / 现象体验、模型激活 / 机械响应、协作任务 / 群体完成。

### 科技、工业与原理展示

常用动作：

- 拆解或扫描部件，显示内部结构
- 转动旋钮或手轮，控制参数变化
- 操作实验台，观察结果生成
- 对比材料样本，触发性能反馈
- 站入测试区，看到数据可视化
- 多人协作完成生产或系统流程

适合反馈：剖面动画、参数曲线、部件点亮、机械响应、屏幕结果、声音或震动反馈。

优先机制：模型激活 / 机械响应、参数实验 / 变量反馈、尺度转换 / 微观宏观、影像映射 / 层级显现。

### 品牌、企业与创新能力

常用动作：

- 选择需求场景，匹配产品或解决方案
- 放置产品样本，触发应用场景展开
- 连接两个技术点，生成创新路径
- 操作材料或工艺台，看到性能差异
- 扫描产品，墙面展开从研发到应用的链路
- 多人围绕模型讨论，结果同步到大屏

适合反馈：应用地图、解决方案网络、场景投影、产品剖面、灯光连接线、客户旅程图。

优先机制：实物操作 / 道具触发、多视角 / 选择判断、创造输出 / 留下结果、角色进入 / 任务身份。

### 交通、速度与运动

常用动作：

- 投球、骑行、跑步、踩踏或推动实体件
- 站入风洞或速度测试区
- 操控模拟驾驶、航线或路径
- 调节速度参数，观察轨迹变化
- 多人协作完成调度或起降
- 身体动作引导虚拟物体移动

适合反馈：轨迹线、速度曲线、风场光束、地面路径、动态灯光、机械平台或屏幕模拟。

优先机制：身体进入 / 现象体验、参数实验 / 变量反馈、空间路径 / 站位叙事、协作任务 / 群体完成。

### 文化、非遗与工艺

常用动作：

- 触摸材料样本，看到工艺步骤
- 拼合纹样、图案或结构片段
- 模拟拉坯、编织、打磨、拓印、压印
- 选择工具，触发工序变化
- 听声音辨识工艺场景
- 把个人选择生成一件虚拟作品

适合反馈：步骤投影、纹样生成、材料剖面、声音层、作品上墙、实体道具响应。

优先机制：实物操作 / 道具触发、参数实验 / 变量反馈、影像映射 / 层级显现、创造输出 / 留下结果。

### 社会议题与价值判断

常用动作：

- 做选择或投票，看到不同立场并置
- 站到不同位置，听到不同人物声音
- 翻开档案或证言，显现隐藏信息
- 连接事件、人物和影响后果
- 多人协作完成判断，结果不是唯一答案
- 通过空间压迫、距离或光线变化感受议题

适合反馈：多声道声音、分歧路径、证言墙、关系网络、灯光分区、可变化文本片段。

优先机制：多视角 / 选择判断、空间路径 / 站位叙事、声音定位 / 口述记忆、创造输出 / 留下结果。

### 儿童与教育探索

常用动作：

- 低位触摸、放置、旋转、踩踏
- 用身体进入现象：风、水、光、声音、速度
- 通过游戏任务完成观察、比较、分类
- 让成人陪伴观看，不抢占主动作
- 给开放结果，而不是只答对错
- 把抽象知识变成孩子手够得到、眼睛看得到的变化

适合反馈：低位操作台、地面点位、墙面生成、灯光反馈、声音反馈、可动模型、亲子尺度关系。

优先机制：角色进入 / 任务身份、身体进入 / 现象体验、实物操作 / 道具触发、参数实验 / 变量反馈。

## 动作选择提醒

不要把“点击屏幕”当成默认互动。优先选择能表达概念的身体动作、实物动作、实验动作、站位路径、声音触发、多人协作或创造输出，再决定屏幕、投影、灯光、声音等载体。

## 提案输出格式

当用户还没到生图状态时，用这个格式：

```text
我会先把这个主题转成可画的互动方向。

方向 1：{名称}
主机制：{从玩法机制库选 1 个}
参与动作：{人怎么做}
触发：{触发点}
反馈：{反馈在哪里发生}
结果：{看到/感受到什么}
适合画面：{一句构图判断}

方向 2：...
方向 3：...

我建议先画方向 {n}，因为它的人物动作和反馈关系最清楚。
```

如果用户明确要求扩展想法，输出 2-3 个方向；如果用户只是玩法不清但明显想尽快出图，只输出 1 个最强方向，避免拖慢生图。

## 禁止

- 不要把抽象主题直接变成漂亮空间效果图。
- 不要先决定用屏幕、投影、灯光，再倒推玩法。
- 不要把任何参考案例当成可复制模板。
- 不要堆砌行业名、装置名或不可验证的出处说明。
- 不要输出一堆拆解，而忘了最终要形成一张能看懂玩法的图。
